# Work Queue Requirements

## R1

Build a small internal web tool for a support team to record incoming work requests and assign each request to one team member.

## R2

Each request must have a title, a short description, a requester name, an assignee, and one status: New, In Progress, or Complete.

## R3

Team members can create requests, view all requests, update an assignee, and update the request status.

## R4

Supervisors can view a simple count of requests by status.

## R5

The first release does not include email notifications, customer accounts, file attachments, or automated assignment.
