# Golden Documentation Analysis

The reference documents separate scope, behavior, technical decisions, API conventions, implementation patterns, verification, and completion checks. They use explicit assumptions rather than silently filling requirement gaps; name one source of truth per concern; repeat consistent section ordering; and use cross-document references to avoid duplicated authority.

The reusable method is the separation of responsibilities, not the reference project's stack, entities, roles, workflows, vendors, or naming. `AGENTS.md` and `PROMPTS.md` demonstrate that implementation governance should be static and consume project knowledge rather than embed it.
