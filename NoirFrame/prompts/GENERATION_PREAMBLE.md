# Generation Preamble

## Required Inputs

Read the requirements source, `FRAMEWORK_SPEC.md`, `FRAMEWORK_MANIFEST.md`,
`PLACEHOLDER_SPEC.md`, `GENERATION_GUIDE.md`, `VALIDATION.md`, the target
template, initialized traceability register, and named upstream documents.
When the target is `TRACEABILITY.md`, initialize the requirement register
directly from the requirements source before applying this preamble.

## Required Output

Produce the target document with valid metadata, only target-owned canonical
claims, stable IDs, required traceability links, explicit uncertainty, and no
unresolved final placeholders.

## Execution

Follow the Universal Generation Contract. Generate only target-owned
information, preserve unchanged IDs, and stop for blocking questions. Do not
invent project facts, architecture, access control, workflows, integrations,
or technical choices. Complete draft, mechanical validation, quality review,
one permitted repair/review pass, and revalidation before final output.
